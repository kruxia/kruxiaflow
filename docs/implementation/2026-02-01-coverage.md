### Testing Strategy for CLI Commands

CLI commands are challenging to unit test because they orchestrate I/O (database connections, server startup, HTTP calls). The approach:

1. **Extract testable logic** — factor pure validation, configuration assembly, and output formatting into separate functions that can be tested in isolation
2. **Test validation paths** — parameter validation, environment variable parsing, configuration merging
3. **Test output formatting** — status display, health check formatting, error messages
4. **Mock-based tests** — where feasible, test command logic against mocked services
