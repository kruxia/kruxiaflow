## Activity Extensibility Model

### Three-Tier Activity System

#### Tier 1: Built-in (Compiled)
- Ships with binary
- Zero latency, maximum performance
- Covers 90% of use cases
- Rust implementations

#### Tier 2: Plugin Activities (WASM)
```rust
// Users can write custom activities in any language that compiles to WASM
// Loaded at startup, near-native performance
#[activity]
async fn custom_ml_inference(input: Value) -> Result<Value> {
    // Custom ML model inference
}
```

#### Tier 3: External Workers
- For specialized/heavy compute
- GPU-intensive ML workloads  
- Legacy system integration
- Language-specific requirements

## Configuration Philosophy

**Zero Configuration for Common Cases:**
```yaml
# This COMPLETE workflow runs with zero external dependencies
workflow: customer_support
activities:
  - key: classify_ticket
    type: llm_complete  # Built-in
    parameters:
      prompt: "Classify this support ticket..."
      
  - key: search_knowledge
    type: vector_search  # Built-in
    parameters:  
      query: "{{classify_ticket.category}}"
      
  - key: generate_response
    type: llm_complete  # Built-in
    parameters:
      prompt: "Generate a response..."
      
  - key: send_response  
    type: email_send  # Built-in
    parameters:
      to: "{{workflow.input.customer_email}}"
```
