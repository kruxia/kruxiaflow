# Workflow

- id — unique for this workflow run
- activities: 
    - step — the current step for the workflow
    - (node) key — unique identifier for this activity node in this position in the workflow. Used in Edge definitions
    - namespace.name — uniquely identify which activity under which service
    - workflow_id — which workflow run this activity is part of
    - scheduled_at — optional time after which the activity can run
    - parameters — inputs: some hard-coded, some variable based on context:
        - previous activities' outputs
    - outputs — values produced by the activity
    - settings, e.g.:
        - retry logic on timeout or failure
        - timeout ttl — how long the activity has to complete
        - budget restrictions
    - edges: 
        - preceding_key: key for an activity that precedes this one; or 
        - following_key: key for an activity that follows this one 
        - conditions: e.g.,
            - preceding has run < max times
            - preceding has now succeeeded (default) or now failed
            - edge has activated < max times
